convergence planning loop ───────────────────────────────────────────── round 1 → spawn: domain-planner → findings spawn: data-planner → findings + cross-layer impact spawn: pres-planner → findings round 2 → spawn: app-planner → impact resolved orchestrator: converged → writes plan.md ───────────────────────────────────────────── ▶ Claude shows you the plan. You approve or redirect. Nothing is built until you say so.
The approval gate is the quality argument. You sign off on the plan. Claude builds exactly what you approved — no inline decisions, no layer-assignment guesses mid-execution.
Step 3 — feature-worker

Layer by layer. Architecture enforced. No surprises.

plan.md approved ✓ → feature-worker executing ──────────────────────────────────────────────────────────── Domain → entities, use cases, repository interfaces Add force: Bool to UseCaseType + all implementations Data → DTOs, mappers, datasources, repository impls Migrate 3 datasources to new pattern Remove unused cache layer in Connection.swift Pres → StateHolder, screens, components Propagate force: Bool from presentation down App → DI, routing, module wiring Update dependency graph for refactored use cases ──────────────────────────────────────────────────────────── ## Output → verified paths returned (Glob + Grep checked)
Each artifact is created against platform reference docs — Swift protocols, patterns, and naming conventions are enforced by the skills the worker calls, not left to inference.
